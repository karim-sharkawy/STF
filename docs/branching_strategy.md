The four branches are:
* `main`
* `clusters`
* `cosmos`
* `entropy` *(experimental branch)*

I didn't wanna make them longer because that's a pain to type each time

### `clusters`

`clusters` is related to MVP 2, which is really about:

* semantic grouping
* interactive gravity
* emergent structure

So `clusters` feels natural and memorable.

### `cosmos`

Related to MVP 3 because it turns the project into an actual interactive universe:

* realtime rendering
* camera movement
* hover labels
* UI controls
* polished demo

`cosmos` captures that much better than something like `realtime-visualization`.


### `entropy`

Perfect name for the weird experimental branch right :)? This goes into

* heat death
* particle decay
* word birth/death
* lingering semantic scars

Fits the simulation theme extremely well.

---

So the final structure becomes:

```
main
 └── clusters
       └── cosmos
              └── entropy
```

Which is clean, memorable, and aesthetically aligned with the project itself.
